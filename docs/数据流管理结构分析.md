# 数据流管理结构分析

## 整体架构概览

数据流通过 5 个核心组件编排，形成一个 **"UI 选择 → 异步计算 → 注册 → 渲染"** 的标准管线：

```
┌─────────────────────────────────────────────────────────────────────┐
│                    应用层 (src/)                                     │
│                                                                     │
│  CustomDialog (工具/插件)                                            │
│    │ m_cloudtree, m_cloudview, m_progress                           │
│    │ 通过 createDialog<T>() 注入                                     │
│    ↓                                                                │
│  ┌──────────────────────────────────────────┐                       │
│  │  CloudTree (编排中心)                      │                      │
│  │  ├─ CloudRegistry   (数据注册表)           │                      │
│  │  ├─ CloudIOController (文件 I/O 编排)     │                       │
│  │  └─ ProgressManager  (异步/进度管理)       │                      │
│  └───────────┬───────────────┬──────────────┘                       │
│              ↓               ↓                                      │
│     CloudView (渲染)      FileIO (QThread)                          │
└─────────────────────────────────────────────────────────────────────┘
              ↓               ↓
┌─────────────────────────────────────────────────────────────────────┐
│                    库层 (libs/)                                       │
│  pw_core (Cloud)    pw_algorithm (静态方法)    pw_io (FileIO)        │
└─────────────────────────────────────────────────────────────────────┘
```

## 一、组件注入机制

所有工具/插件继承 `CustomDialog`，通过 `createDialog<T>()` 模板工厂获得依赖注入（`customdialog.h:92-173`）：

```
MainWindow 调用 createDialog<Filters>(...)
  → 注入 CloudView*    (m_cloudview)    // 当前活跃视窗
  → 注入 CloudTree*    (m_cloudtree)    // 编排中心
  → 注入 Console*      (m_console)      // 日志
  → 注入 ProgressManager* (m_progress)  // 从 CloudTree 获取
  → 注入 ViewportManager* (m_viewport_mgr)  // 多视窗管理
  → 调用 dlg->init()
```

工具通过 `m_cloudtree` 访问所有数据操作，通过 `m_cloudview` 直接操作渲染，通过 `m_progress` 执行异步任务。

## 二、文件加载流 (Load Path)

```
用户点击"打开" → CloudTree::addCloud()
    → CloudIOController::addCloud()
        → QFileDialog 选文件
        → ProgressManager::showProgress()
        → QMetaObject::invokeMethod(FileIO, "loadPointCloud", QueuedConnection)
            ↓ [切换到 FileIO 所在的 QThread]
        FileIO::loadPointCloud(filepath)
            → 读取 Header → 流式加载（每批 50 万点）→ 构建八叉树 → 生成 LOD
            → emit loadCloudResult(success, cloud, mesh, time)
            ↓ [信号回到主线程]
        CloudIOController::onLoadCloudResult()
            → emit cloudLoaded(cloud, mesh, parent, time)
            ↓
        CloudTree::handleCloudLoaded()
            → CloudTree::insertCloud(cloud, parent)
                → cloud->update()                    // 更新内部状态
                → makeUniqueName()                    // ID 去重
                → addItem(parent, cloud->id())        // 创建 TreeWidgetItem
                → setItemViewports()                  // 绑定到活跃视窗
                → m_registry->registerCloud(item, cloud)  // 注册到 Registry
                → CloudView::addPointCloud(cloud)     // 渲染
                → CloudView::zoomToBounds()           // 自动聚焦
                → emit cloudInserted(cloud)           // 通知 Python Bridge
```

**关键点**：
- FileIO 运行在独立 QThread，通过信号/槽跨线程回传
- CloudRegistry 维护 `item → Cloud::Ptr` 和 `id → item` 的双向映射
- 每个 Cloud 的 ID 必须全局唯一，冲突时自动重命名

## 三、算法处理流 (Process Path)

以 CSFPlugin 为例（`csfplugin.cpp:35-88`），这是最典型的处理流：

```
1. 获取输入数据
   CSFPlugin::init()
     → m_cloud = m_cloudtree->getSelectedClouds().front()
     // 从 CloudTree 获取用户选中的 Cloud::Ptr

2. 收集参数
   CSFPlugin::onApply()
     → 读取 UI 控件值 (smooth, thresh, res, rigidness, iter)

3. 异步执行算法
   m_progress->runAsync<CSFResult>(
       "Running CSF...",
       [cloud, params...](atomic<bool>& cancel, ProgressCallback progress) {
           // [后台线程]
           return CSFFilter::apply(cloud, ...);  // 纯算法，零 VTK
       },
       [=](const CSFResult& result) {
           // [主线程回调]
           ... 处理结果 ...
       }
   );

4. 结果处理（主线程回调中）
   → 设置 result cloud 的 ID（cloud->id() + "_ground" / "_off_ground"）
   → 调用 cloud->makeAdaptive() 配置自适应八叉树
   → cloudtree->addResultGroup(originCloud, results, groupName)
       → 在 originCloud 的父节点下创建 Group 节点
       → 清除同名旧结果组（防重复）
       → insertCloud() 逐个插入结果点云
       → 自动隐藏原始点云（取消勾选）
       → resetCamera() 聚焦到结果
```

**数据所有权**：算法返回新的 `Cloud::Ptr`（智能指针），原始 Cloud 不被修改。结果 Cloud 的所有权通过 `CloudRegistry::registerCloud()` 转移到注册表。

## 四、滤波预览/应用流 (Preview-Apply Path)

Filters 工具有独特的预览→添加→应用三级流程（`filters.cpp:224-443`）：

```
Preview（预览）:
  Filters::preview()
    → cloud = m_cloudtree->getSelectedClouds()[i]
    → 构建 lambda: [=]() { return pw::Filters::PassThrough(cloud, ...); }
    → runFilter(filterFn, show_progress)
        → QtConcurrent::run(filterFn)       // 后台执行
        → QFutureWatcher 监听完成
        → handleFilterResult(result)
            → cloud->setId(id + "-filter")  // 临时 ID
            → m_cloudview->addPointCloud(cloud)   // 仅渲染，不入树
            → m_filter_map[id] = cloud       // 缓存到本地 map

Add（添加为兄弟）:
  Filters::add()
    → 从 m_filter_map 取出预览结果
    → cloudview->removePointCloud()         // 移除预览
    → cloud->setId("filtered-" + id)        // 正式 ID
    → cloudtree->insertCloud(cloud, item, Sibling)  // 入树+注册+渲染

Apply（就地替换）:
  Filters::apply()
    → 从 m_filter_map 取出预览结果
    → cloudview->removePointCloud()         // 移除预览
    → cloudtree->updateCloud(cloud, new_cloud)
        → cloud->swap(*new_cloud)           // 原地替换数据
        → cloud->update()                   // 重建内部结构
        → cloudview->addPointCloud(cloud)   // 重新渲染
```

**关键点**：
- 预览结果存在工具的 `m_filter_map` 中，**不在 CloudRegistry** 中
- 只有 Add/Apply 才将结果纳入 Registry 管理
- `updateCloud()` 使用 swap 原地替换，保持 TreeWidgetItem 和 ID 不变

## 五、文件保存流 (Save Path)

```
CloudTree::saveCloudFile(cloud, filepath)
  → CloudIOController::saveCloudFile(cloud, filepath)
      → ProgressManager::showProgress("Saving...")
      → QMetaObject::invokeMethod(FileIO, "savePointCloud", QueuedConnection)
          ↓ [切换到 FileIO QThread]
      FileIO::savePointCloud(cloud, filepath)
          → 根据扩展名选择格式写入
          → emit saveCloudResult(success, path, time)
          ↓ [回到主线程]
      CloudIOController::onSaveCloudResult()
          → emit saveComplete(success, path, time)
```

## 六、Python 脚本流 (Script Path)

```
PythonConsole / PythonEditor
  → PythonWorker (QThread, 持有 GIL)
      → 执行用户 Python 脚本
      → 通过 pybind11 绑定调用 ct 模块
          → bind_cloud_mgmt: ct.loadCloud() / ct.saveCloud()
          → bind_filters: ct.PassThrough() / ct.VoxelGrid()
          → ... 16 个绑定模块
      → PythonBridge (信号桥接)
          → 内部持有 CloudRegistry（线程安全注册表）
          → 信号通知主线程更新 UI
```

## 七、数据流关键模式总结

| 模式 | 描述 |
|------|------|
| **异步执行** | 所有耗时操作通过 `ProgressManager::runAsync()` 或 `QtConcurrent::run()` 在后台执行，主线程通过回调处理结果 |
| **注册中心** | `CloudRegistry` 作为唯一的数据注册表，维护 item↔Cloud 和 id→item 双向映射 |
| **不可变输入** | 算法不修改输入 Cloud，总是返回新的 `Cloud::Ptr`；原地替换通过 `updateCloud()` 的 swap 实现 |
| **挂载策略** | 5 种挂载策略（Auto/Sibling/Child/Group/Root）控制结果在树中的位置 |
| **ID 全局唯一** | `makeUniqueName()` 确保每次 insert 前检查冲突 |
| **脏标记渲染** | CloudBlock 的 `m_is_dirty` 标记控制渲染缓存更新，避免全量重建 |
| **线程边界** | FileIO 在独立 QThread；算法在 QtConcurrent 线程池；Python 在 Worker QThread + GIL；所有 UI 操作回到主线程 |

## 八、核心数据流向图

```
                    ┌─────────────────┐
                    │  用户交互        │
                    │ (选点云/设参数)  │
                    └────────┬────────┘
                             ↓
                    ┌─────────────────┐
                    │ CustomDialog     │
                    │ m_cloudtree→     │
                    │   getSelected()  │──→ Cloud::Ptr (智能指针，只读引用)
                    └────────┬────────┘
                             ↓
              ┌──────────────┼──────────────┐
              ↓              ↓              ↓
      ┌──────────────┐ ┌────────────┐ ┌──────────────┐
      │ FileIO       │ │ Algorithm  │ │ Python       │
      │ (QThread)    │ │ (线程池)    │ │ (Worker线程) │
      │ 流式加载/保存 │ │ 静态方法    │ │ pybind11绑定 │
      └──────┬───────┘ └─────┬──────┘ └──────┬───────┘
             ↓               ↓               ↓
        Cloud::Ptr      Result{Cloud,     ct.Cloud 对象
        (新构建)         time_ms}
             ↓               ↓               ↓
             └───────────────┼───────────────┘
                             ↓ (回到主线程)
                    ┌─────────────────┐
                    │ CloudTree        │
                    │  insertCloud()   │──→ CloudRegistry.registerCloud()

                    │ CloudView        │

                    │  (VTK 渲染)      │      (SSE遍历 + 视锥剔除 + LOD)
                    └─────────────────┘
```
